# GoodMonk D2C Command Center

**Monitor · Detect · Alert · Optimize**

An internal website health & performance monitoring platform for GoodMonk
(Superfoods Valley). It runs a robot-customer journey against the live store,
diagnoses *why* the site is slow with Google PageSpeed Insights, logs everything
to a Google Sheet, emails alerts when something breaks, and renders an executive
dashboard you can host for free.

This is **not** a Shopify storefront or redesign — it is the ops/management
console that watches the storefront.

---

## Two connected systems

**System 1 — Critical Break Detection** ("Is anything on fire?")
A Playwright robot customer walks Homepage → Collection → H50+ PDP → FNM PDP →
Add to Cart → Checkout init every 15 minutes. Any 4xx/5xx, failed step, or
>10s load fires a Critical alert. It never completes a real payment.

**System 2 — Performance Diagnostics** ("Why is it slow, and what do we fix?")
PageSpeed Insights runs hourly on 6 pages (mobile + desktop), capturing Core Web
Vitals and the **top 3 specific opportunities per page** (e.g. "1.9MB hero image
not compressed") so the dev team gets a diagnosed issue, not a vague complaint.

Both feed a Google Sheet (history), Gmail alerts (with **de-duplication and
recovery messages** to prevent alert fatigue), and a static dashboard.

## Deploy in 6 steps

### 1. Create the GitHub repository
Create a repo and upload these files (or `git push`). Done.

### 2. Add a PageSpeed Insights API key
Google Cloud Console → enable **PageSpeed Insights API** → create an API key.
Add it as repo secret `PAGESPEED_API_KEY`.

### 3. Add Gmail credentials
Use a Gmail account with 2FA on, then create an **App Password**
(Google Account → Security → App passwords). Add repo secrets:
`GMAIL_USER`, `GMAIL_APP_PASSWORD`, `ALERT_RECIPIENTS` (comma-separated).

### 4. Connect the Google Sheet
- Create a blank Google Sheet; copy its ID from the URL.
- Google Cloud → create a **service account** → download its JSON key.
- **Share the Sheet** with the service account's email (Editor).
- Add repo secrets `GOOGLE_SHEET_ID` and `GOOGLE_SERVICE_ACCOUNT_JSON`
  (paste the entire JSON as the value). The tabs `Health_Checks`,
  `Performance`, `Alerts` are created automatically on first run.

Also add `GM_BASE_URL` (e.g. `https://goodmonk.in`) and confirm the product
paths in `monitoring/config.py` match the live URLs.

### 5. Turn on the schedules
The two workflows in `.github/workflows/` run automatically (every 15 min and
hourly). Trigger them once manually from the **Actions** tab to seed live data.
Each run commits refreshed `dashboard/data/*.json` back to the repo.

### 6. Upload the dashboard to InfinityFree
Upload the contents of `dashboard/` to your InfinityFree `htdocs`. Then open
`app.js` and point it at your committed data:

```js
const DATA_BASE = 'https://raw.githubusercontent.com/<user>/<repo>/main/dashboard/data/';
```

Visit `login.html`, sign in with **admin / GMSFV**, and you're live.

## Run locally (Windows 11 / Python 3.12+)

```bash
cd monitoring
python -m venv .venv && .venv\Scripts\activate     # Windows
pip install -r requirements.txt
python -m playwright install chromium
copy .env.example .env                              # then fill it in

python seed_sample_data.py     # optional: demo data for the dashboard
python monitor.py              # run the robot journey once
python run_performance.py      # run PageSpeed once
python json_generator.py       # rebuild dashboard JSON
```

Preview the dashboard with a local server (needed for `fetch` to work):

```bash
cd dashboard
python -m http.server 8080      # open http://localhost:8080/login.html
```

Without Google Sheets configured, the logger writes to
`dashboard/data/_sheet_fallback/` so everything still runs.

## Alerting tiers

The brief specifies 3 tiers. Email (Tier 3) is fully implemented with Gmail SMTP,
including de-duplication (one alert per `ALERT_DEDUP_MINUTES` window), escalation
notes for sustained outages, and **resolved** messages on recovery. Tier 2
(WhatsApp) and Tier 1 (phone/Twilio) are wired as clearly marked hooks in
`alerts.py` — they require paid services, so enable them when ready.

**Alert email template**

```
Subject: GOODMONK CRITICAL ALERT
Severity      : Critical
Issue         : Checkout initiation failed (HTTP 503)
Affected Page : Checkout
Time          : 2026-06-24 09:14:32 IST
Action        : Verify Shopflo/Shopify checkout and payment gateway immediately.
```

## Thresholds (from the brief)

| Signal | Healthy | Warning | Critical |
|---|---|---|---|
| Page load | < 4s | 4–10s | > 10s, or any 4xx/5xx / failed step |
| LCP | < 2.5s | — | benchmark miss flagged |
| CLS | < 0.1 | — | benchmark miss flagged |
| INP | < 200ms | — | benchmark miss flagged |
| TTFB | < 0.8s | — | benchmark miss flagged |
| Page weight | — | — | flag > 2.5MB mobile |
| Image weight | — | — | flag > 60% of total |

Add-to-cart / checkout escalate to Critical after **2 consecutive** failures.

## Security note

The dashboard login is a lightweight client-side gate (admin / GMSFV) suitable
for an internal tool on static hosting — it keeps casual visitors out but is not
server-side auth. For stronger protection, add `.htaccess` HTTP basic auth on
InfinityFree in front of the dashboard.
